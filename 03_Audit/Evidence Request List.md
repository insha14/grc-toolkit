# Evidence Request List

## Overview
An evidence request list outlines the documentation, screenshots, reports, and records that auditors require to verify whether controls are designed and operating effectively. Providing clear and organized evidence is essential for completing internal audits, SOC 2 assessments, ISO certifications, and ITGC reviews. This list summarizes commonly requested evidence across key control areas to support audit readiness and streamline communication between auditors and control owners.

## Purpose of Evidence Requests
Evidence requests help auditors confirm that policies are in place, controls are functioning as intended, and activities such as access reviews, backups, and change approvals are properly documented. Clear evidence reduces audit delays, prevents misunderstandings, and strengthens the reliability of audit conclusions. Well-prepared evidence also demonstrates organizational maturity and strong governance practices.

## Evidence Request Categories

### 1. Access Management Evidence
| Evidence Item | Description |
|---------------|-------------|
| User Access List | Export of all active users for applications or systems under review. |
| Terminated User Report | Documentation showing timely removal of access for departed employees. |
| Privileged Access Listing | List of admin or elevated accounts with justification. |
| MFA Configuration Screens | Evidence showing MFA is enforced for privileged and general users. |
| Access Review Records | Completed access review forms or screenshots showing approval. |

### 2. Change Management Evidence
| Evidence Item | Description |
|---------------|-------------|
| Change Request Tickets | Documentation of approved changes including approvals and timestamps. |
| Testing Evidence | Screenshots or logs confirming testing was performed before deployment. |
| Emergency Change Logs | Documentation showing emergency changes were reviewed post-implementation. |
| Deployment Records | Evidence that changes were deployed following the approved process. |

### 3. IT Operations Evidence
| Evidence Item | Description |
|---------------|-------------|
| Backup Reports | Logs showing that backups were performed successfully. |
| Restore Testing Evidence | Records demonstrating that backup data was tested for recovery. |
| Log Review Screenshots | Evidence that logs are reviewed regularly by IT or security teams. |
| Incident Tickets | Documentation of reported security incidents and their resolutions. |
| Monitoring Dashboards | Screenshots of SIEM tools, alerts, or monitoring platforms. |
