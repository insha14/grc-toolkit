
# Onboarding Security Checklist

## Overview
The Onboarding Security Checklist provides a structured process for ensuring that new employees, contractors, or third-party users are granted appropriate and secure access to organizational systems. This checklist supports secure provisioning, reduces the risk of misconfigured accounts, and ensures all onboarding activities align with security and compliance requirements.

## Objective
The objective of this checklist is to help IT, HR, and security teams coordinate onboarding tasks so new users begin with secure access, proper training, and clear expectations regarding acceptable use and responsibility.

## Checklist Items

### HR Coordination
- Confirm employee identity and employment status.
- Verify job role, department, and start date.
- Ensure HR has submitted a formal access request for required systems.
- Document whether the user needs remote access, VPN, or privileged access.

### Account Provisioning
- Create primary user account in the identity management system.
- Assign appropriate access based on role (least privilege).
- Enable multi-factor authentication (MFA).
- Create or provision email account.
- Configure default security settings (password policy, device compliance).
- Provide initial credentials following secure distribution guidelines.

### System and Application Access
- Grant access only to approved applications and resources.
- Assign role-based or group-based permissions.
- Document any privileged or admin access with justification.
- Validate access accuracy with the hiring manager.

### Device Provisioning
- Issue approved laptop or workstation.
- Install required security tools (EDR, VPN, patching tools, encryption).
- Apply standard system configuration and hardening guidelines.
- Enable automatic updates and monitoring.

### Policy Acknowledgements
- Provide Acceptable Use Policy.
- Provide Information Security Policy.
- Provide Data Handling or Privacy Policy (if applicable).
- Record signed acknowledgements or LMS completion.

### Security Training
- Assign mandatory security awareness training modules.
- Assign phishing awareness training (if applicable).
- Track training completion in LMS or HR system.

- Hiring manager verifies all access is correct.
- Security team confirms MFA activation, endpoint health, and compliance.
